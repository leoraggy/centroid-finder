# Centroid-finder 
A Spring Boot REST API for managing video files, generating thumbnails, and launching asynchronous centroid-processing jobs.

## Processor Section
## Example: Build and run the video processor
The `processor` module can produce centroid summaries from video frames using `VideoSummaryApp`.

1. Build the shaded JAR with Maven:

```bash
cd processor
mvn package
```

2. Run the processor with a video file, an output CSV path, a hex target color, and a distance threshold:

```bash
java -jar target/centroidfinder-1.0-SNAPSHOT-jar-with-dependencies.jar "path/to/video.mp4" "output/groups.csv" FFA200 164
```

3. The application will create the output directory if needed and write a CSV file containing rows like:

```csv
group_size,centroid_x,centroid_y
```

4. If the output CSV already exists, it will be overwritten and a warning will be printed.

> Note: `VideoSummaryApp` processes every 60th frame and writes only the largest detected group per processed frame.


 
## Server Section

## Example: Run the server

```bash
cd server
./mvnw spring-boot:run
```




## Features

- List available `.mp4` videos
- Generate video thumbnails using FFmpeg
- Start asynchronous video processing jobs
- Track processing job status
- Export centroid detection results to CSV

```properties

```

This directory should contain the MP4 files that will be processed.

# Endpoints

## 1. List Videos

Returns all MP4 files in the configured video directory.

### Request

```http
GET /api/videos
```

### Response

```json
[
  "video1.mp4",
  "video2.mp4",
  "sample.mp4"
]
```

### Error Response

```json
{
  "error": "Configured path is not a valid directory"
}
```

---

## 2. Generate Thumbnail

Creates a JPEG thumbnail from the first second of a video.

### Request

```http
GET /api/thumbnail/{filename}
```

### Example

```http
GET /api/thumbnail/sample.mp4
```

### Success Response

Content-Type:

```text
image/jpeg
```

Returns raw image bytes.

### Error Response

```json
{
  "error": "Video file not found: sample.mp4"
}
```

---

## 3. Start Processing Job

Launches an asynchronous centroid detection process.

### Request

```http
POST /api/process/{filename}?targetColor=red&threshold=50
```

### Parameters

| Parameter | Type | Required | Description |
|------------|--------|----------|-------------|
| targetColor | String | Yes | Target object color |
| threshold | Integer | Yes | Color threshold value |

### Example

```http
POST /api/process/sample.mp4?targetColor=red&threshold=50
```

### Success Response

HTTP Status:

```http
202 Accepted
```

Body:

```json
{
  "jobId": "8bce1c75-cf0b-4b06-a46d-3cddf9af7f31"
}
```

### Error Response

```json
{
  "error": "Missing targetColor or threshold query parameter."
}
```

---

## 4. Check Job Status

Retrieves the status of a processing job.

### Request

```http
GET /api/process/{jobId}/status
```

### Example

```http
GET /api/process/8bce1c75-cf0b-4b06-a46d-3cddf9af7f31/status
```

### Processing Response

```json
{
  "status": "processing"
}
```

### Completed Response

```json
{
  "status": "done",
  "outputFile": "C:\\videos\\sample.mp4.csv",
  "error": null
}
```

### Error Response

```json
{
  "status": "error",
  "outputFile": null,
  "error": "Processor exited with error code: 1"
}
```

### Job Not Found

```json
{
  "error": "Job ID not found"
}
```

# Processing Workflow

```text
Client
  |
  v
POST /api/process/{filename}
  |
  v
Generate Job ID
  |
  v
Launch Processor JAR
  |
  v
Process Video Frames
  |
  v
Generate CSV Output
  |
  +---- Success --> status = done
  |
  +---- Failure --> status = error
```

---

# Output

Successful processing generates a CSV file:

```text
<video filename>.csv
```

Example:

```text
sample.mp4.csv
```

The CSV contains centroid detection results generated by the processor application.
